Art Office
Manage Art's identity, memory, skills, and MCP connections.

Identity editor Phase 1

Edit Art's core files — SOUL.md, AGENTS.md, TOOLS.md, HEARTBEAT.md, USER.md — via structured forms. Raw markdown always available.

Memory viewer Phase 1

View MEMORY.md by section. Trim controls: delete section, truncate to last N entries, clear all. Auto-alerts when memory exceeds threshold.

Skills browser Phase 1

Browse, add, edit, and remove Art's skill files.

Model + MCP Phase 1

Switch Art's provider and model without restarting the service. MCP server connection list: add, remove, test. Connections auto-apply on save.

Lab
Build skill-sets. Define identity, add skills, test in sandbox, tag a version, spawn.

Workspace editor Phase 2

Create from _template or clone from remote. Edit SOUL, AGENTS, TOOLS, HEARTBEAT, USER, skills, MCP config, knowledge inbox.

Sandbox test panel Phase 2

Load a workspace into an ephemeral container, chat with it, iterate. Each Lab tab runs its own isolated container. Promote to spawn when behaviour is correct.

Version control Phase 4

Tag workspace snapshots with git. Browse history, diff any two versions line by line, restore, push to remote for collaboration.

Agents
Registry of spawned production agents. Start, stop, delete, chat.

Agent registry Phase 2

Each spawned agent runs in its own container with its own workspace, knowledge base, and port allocation. Start, stop, delete from here. Click an agent to open a proxied chat in a new tab.

Teams
Build multi-agent teams with a shared knowledge center.

Team builder Phase 3

Assign spawned agents to a team. Set an orchestrator. Connect a shared knowledge library. Members can query the shared knowledge base at runtime.

Arena
Run multi-agent sessions. Observe turn by turn. Replay any past session.

Session runner Phase 3

Pick a mode: Direct (two agents debate or collaborate), Orchestrated (one agent leads, delegates), or Swarm (any agent hands off to any other mid-turn). Set a goal, start, observe.

Replay Phase 3

Every session turn is stored. Load any past arena session and step through it turn by turn.

Benchmark
Evaluate agent quality. Compare versions, models, and judges.

Task set editor Phase 4

Create task sets via form or ask Art to generate them from the workspace's identity files. Each task has an input and evaluation criteria. Task sets are git-versioned alongside the workspace.

Benchmark runs Phase 4

Select workspace + version tag + model + judge model. Run. LLM-as-judge scores each response 0–10 with a reason. Compare across versions and models in the results chart.

Settings
Provider profiles, default model, app configuration.

Provider profiles Phase 1

Add, edit, and test LLM provider connections — Ollama, LM Studio, xAI, OpenAI, custom. Each profile declares its capabilities (text, tools, image, embeddings). All profiles testable with one click.

App config Phase 1

Default model, memory alert threshold, cost indicator soft/hard limits, Lab pre-warm pool size, log level.
